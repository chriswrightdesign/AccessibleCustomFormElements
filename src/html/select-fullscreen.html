Full screen select box: Do not use in a real project

As Luke Wroblewski calls select boxes the UI of last resort, I wanted to make an alternative which had fallback to the standard select box, most of the solutions I've seen where people have created non-hybrid select boxes have been completely rewriting the solution.

This solution is potentially problematic, it requires you to inject an element above the select box, create an element with mirrored content to the select box and then sync the click to the option in the seleect box.

This will need a lot of testing, and probably some ARIA love.

At the moment this works pretty well on the devices and browsers I've tested (see the readme for the google spreadsheet, however we've lost keyboard control that would be there if this was a simple select element. Our options here are to either - add a listener for keyboard input and assertlive, or when a key is pressed remove the custom stuff and fallback to the standard select box.
